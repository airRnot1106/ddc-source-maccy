INTRODUCTION                                  *ddc-source-maccy-introduction*

This source completes recent clips from Maccy (the macOS clipboard manager).
It reads Maccy's Core Data store directly via `sqlite3` and offers clips
copied within a recent time window as completion candidates. Confirming a
candidate inserts the clip verbatim, including multi-line text.

Filtering and ordering are done in SQL, so this source ships no matcher,
sorter or converter. Configure the consumer with empty filters and let items
pass through unchanged: >vim
	matchers = {}, sorters = {}, converters = {}
<

==============================================================================
INSTALL                                            *ddc-source-maccy-install*

Please install both "ddc.vim" and "denops.vim". `sqlite3` must be on your
PATH, and this source only supports Neovim (it uses |nvim_buf_set_text()| to
expand multi-line clips).

https://github.com/Shougo/ddc.vim
https://github.com/vim-denops/denops.vim


==============================================================================
EXAMPLES                                          *ddc-source-maccy-examples*
>lua
	vim.fn["ddc#custom#patch_global"]("sources", { "maccy" })

	vim.fn["ddc#custom#patch_global"]("sourceOptions", {
	  maccy = {
	    mark = "[Maccy]",
	    matchers = {}, sorters = {}, converters = {},
	    minAutoCompleteLength = 0,
	    isVolatile = true,
	    maxItems = 10,
	  },
	})

	vim.fn["ddc#custom#patch_global"]("sourceParams", {
	  maccy = {
	    recentMs        = 300000,
	    cacheTtlMs      = 2000,
	    types           = { "public.utf8-plain-text" },
	    dbPath          = "~/Library/Containers/org.p0deje.Maccy/Data/Library/Application Support/Maccy/Storage.sqlite",
	    maxByteLength   = 10000,
	    dedupe          = true,
	    excludeAfterUse = true,
	    maxUseCount     = 1,
	  },
	})
<

==============================================================================
PARAMS                                              *ddc-source-maccy-params*

                                            *ddc-source-maccy-param-recentMs*
recentMs	(number)
		How many milliseconds back to include clips. 0 emits
		(almost) nothing; set a large value for effectively
		unlimited.

		Default: 0

                                          *ddc-source-maccy-param-cacheTtlMs*
cacheTtlMs	(number)
		Time-to-live, in milliseconds, of the in-memory query cache.
		Reuses the last result while it is fresh to avoid hammering
		sqlite3 during volatile completion. 0 disables caching.

		Default: 0

                                               *ddc-source-maccy-param-types*
types		(string[])
		Target ZTYPE values to fetch. The practical value is
		["public.utf8-plain-text"]. An empty list emits nothing.

		Default: []

                                              *ddc-source-maccy-param-dbPath*
dbPath		(string)
		Path to Maccy's Storage.sqlite. A leading ~ is expanded with
		$HOME. An empty string emits nothing.

		Default: ""

                                       *ddc-source-maccy-param-maxByteLength*
maxByteLength	(number)
		Maximum byte length of a clip to include (a SQL LENGTH filter).
		A value <= 0 emits nothing.

		Default: 0

                                              *ddc-source-maccy-param-dedupe*
dedupe		(boolean)
		Collapse clips that share the same text, keeping only the
		most recently copied one.

		Default: v:false

                                     *ddc-source-maccy-param-excludeAfterUse*
excludeAfterUse	(boolean)
		Exclude a clip from candidates once it has been used (i.e.
		its completion was confirmed) |ddc-source-maccy-param-maxUseCount|
		times. Re-copying the same text resets its use count. Has no
		effect unless combined with maxUseCount.

		Default: v:false

                                         *ddc-source-maccy-param-maxUseCount*
maxUseCount	(number)
		Number of times a clip may be used before it is excluded, when
		excludeAfterUse is enabled. 0 excludes the clip immediately,
		even before its first use. Ignored unless excludeAfterUse is
		enabled.

		Default: 1


==============================================================================
PHILOSOPHY                                      *ddc-source-maccy-philosophy*

This source is intentionally inconvenient by default. The shipped defaults are
extreme values that emit nothing, forcing you to set every parameter
explicitly. "Not configured" states (`dbPath == ""`, `types == {}`,
`maxByteLength <= 0`) return an empty list without ever launching sqlite3.

Runtime failures (sqlite3 missing, the database unreadable, a malformed
query) are thrown and left to ddc.vim, rather than being swallowed.


==============================================================================
HEALTH                                              *ddc-source-maccy-health*

Run |:checkhealth| ddc-source-maccy to verify that `sqlite3` is on your PATH,
denops.vim is available, and the Maccy database exists and is readable.
